How Lumen was built

A behind-the-scenes look at the system prompt, UI decisions, and roadmap for this AI Study Assistant.

1. System prompt design Prompt Engineering

Lumen is powered by a structured system prompt covering role, scope, constraints, output formatting, and edge-case handling — shown in full below. Key ideas:

  • Role & scope: A patient, encouraging study mentor for college students — strictly scoped to learning, academics, coding help, and productivity.
  • Format-aware replies: The prompt instructs Claude to choose the right output shape — a plan, a quiz, a step-by-step explanation, or a conversational answer — based on the request.
  • Missing information: Instead of blocking on missing details (exam date, subject level), Claude is told to make one reasonable assumption, state it, and proceed — so the student always gets a usable answer.
  • Safety rails: No academic dishonesty (e.g. "write my entire assignment for submission as-is"), no medical/mental-health diagnosis, redirect off-topic requests kindly.

2. UI decisions Design

  • Not a generic chatbot: quick-action cards, a persistent sidebar with history, and a documentation panel give it the shape of a SaaS product rather than a bare chat window.
  • Calm, premium palette: ice white / pearl / soft cloud backgrounds with indigo and slate-blue as primaries, and lavender, sky, mint, peach, and butter as soft accent gradients — never pure black, never neon.
  • Motion with restraint: floating gradient blobs, a cursor-following glow, rising particles, hover-lift on cards, button shine, and a typing indicator create a sense of life without becoming distracting.
  • Accessibility-minded: adjustable font size, light/dark theme toggle, responsive collapsing sidebar, and clear empty/loading/error states.

3. Claude integration Engineering

Messages are sent via fetch() to https://api.anthropic.com/v1/messages with the system prompt and full conversation history on every call, since the API is stateless. No API key is required here — this artifact runs inside the Claude.ai environment, which handles authentication for you.

4. Future improvements Roadmap

  • Real document parsing (PDF/DOCX text extraction) instead of a UI-only upload preview.
  • Persisting conversation history to local storage or an account, across sessions.
  • Spaced-repetition flashcards generated automatically from quiz results.
  • Voice input for hands-free doubt-clearing while revising.
  • Progress dashboard tracking topics studied, quiz accuracy, and weak areas over time.
